# note 有料記事レビュー観点

## 最重要観点
- [ ] 想定ペルソナが「これは自分のための記事だ」と感じられるか
- [ ] 想定ペルソナに対して「買う理由」が明確か
- [ ] 想定ペルソナが購入後に「買ってよかった」と感じる具体性・再現性があるか
- [ ] 想定ペルソナが記事を読んだあと、明日から行動を変えられるか

---

## 構成チェック
- [ ] 導入で「これは自分に必要だ」と確信できるか
- [ ] 問題提起 → 原因 → 手順 → まとめ の流れが自然か
- [ ] 体系化された手順が提供されているか（気づきだけで終わっていないか）
- [ ] ステップが再現可能な粒度で書かれているか
- [ ] 各ステップに「なぜ必要か / やらないとどうなるか / どうやるか」が含まれているか
- [ ] 締めに行動を促す記述があるか
- [ ] 読後に「結局どうすればいいのか」が明確に残るか

---

## 価値チェック
- [ ] 無料記事との差別化ができているか（手順の体系化）
- [ ] 無料記事は「気づき・一部の再現性」、有料記事は「体系化された手順」になっているか
- [ ] 価格（300円 / 500円）に対して、読者が納得する具体性と情報量があるか
- [ ] 読者が「知ってよかった」ではなく「使える」と感じる内容か
- [ ] 実務や日常でそのまま転用できるレベルまで落ちているか
- [ ] 読者が他の記事・他の発信も見たくなる満足度があるか

---

## 売れるかどうかのチェック
- [ ] タイトルだけで「自分ごと化」できるか
- [ ] 導入だけで「続きを読みたい」と思えるか
- [ ] 無料部分だけでも価値を感じられるか
- [ ] 有料部分に入る理由が十分に作れているか
- [ ] 「今買わなくていい理由」より「今読む理由」の方が強いか
- [ ] 読者の痛み・損失・遠回りを明確に言語化できているか

---

## 満足度チェック
- [ ] 購入後に「思ったより薄い」と感じさせないか
- [ ] 購入後に「抽象的で使えない」と感じさせないか
- [ ] 購入後に「無料記事でもよかった」と思われないか
- [ ] 読後に「自分の状況に当てはめて使える」状態になっているか
- [ ] 読後に「次に何をすればいいか」が残っているか

---

## 導線チェック
- [ ] 無料公開ラインの設定が適切か（続きが気になるポイントで切れているか）
- [ ] 無料部分で「使えそう」と感じさせた上で有料に入っているか
- [ ] 次の有料記事への導線があるか（300円 → 500円 の流れ）
- [ ] 関連する無料記事 / 有料記事への回遊導線があるか
- [ ] 導線が売り込みすぎず、自然な流れになっているか

---

## 文体チェック
- [ ] 「弱い断定」をしていないか（例：〜だと思います）
- [ ] 「強い仮説提示」になっているか（例：〜という構造になっている）
- [ ] ペルソナにとって難しすぎる言葉を使っていないか
- [ ] 抽象 → 具体の流れがあるか
- [ ] 説明だけでなく、読者の感情（焦り・あるある・納得）にも触れているか

---

## NGチェック
- [ ] 実績なしに「成功」を語っていないか
- [ ] 誰にでも当てはまる抽象的な表現になっていないか
- [ ] 説明だけで終わっていないか（行動に繋がっているか）
- [ ] 売り込みが前面に出ていないか
- [ ] ペルソナに対して強すぎる断定や誇張をしていないか
- [ ] 自分語りが主役になっていないか

---

## 最終判定
- [ ] このnoteは、想定ペルソナに対して「売れる」と言えるか
- [ ] このnoteは、想定ペルソナに対して「満足してもらえる」と言えるか
- [ ] 95点未満なら、公開せずに修正すべき箇所が明確になっているか
