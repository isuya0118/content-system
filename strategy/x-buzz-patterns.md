## なぜなぜ分析｜なぜバズるのか

| レイヤー | 問い | 答え |
|---|---|---|
| L1 | なぜ止まって読むのか | 1行目が「自己照合」「損失回避」「逆説」のどれかを引き起こすから |
| L2 | なぜそれが機能するのか | 人間は「自分に関係する情報」「損するかも」「思っていたのと違う」に本能的に反応するから |
| L3 | なぜ副業・AIテーマで機能するのか | ①始め方が分からない焦り、②自分だけ遅れている不安、③使っているが本当に使えているか自信がない、の3つが同時に存在するから |
| L4 | なぜ保存・共有されるのか | 「後で自分に使える」「フォロワーに見せたい」の動機が明確に存在するから。単なる「いい話」では保存されない |
| L5 | なぜ1000いいねになるのか | 実用性（ノウハウ系）・共感（体験系）・思考（問い系）のどれかが強ければ保存率が上がり、アルゴリズムが拡散するから |

## バズる構成パターン

| パターン | 構造 | 向いているエンゲージメント |
|---|---|---|
| 箇条書き型 | タイトル1行 → 番号付き5〜7項目 → 締め1行 | 保存・共有 |
| ビフォーアフター型 | Before描写 → 転換点 → After描写 | いいね・拡散 |
| 問いかけ→解答型 | 問い → 観察 → 答え → 読者への示唆 | リプライ |
| 感情開示型 | 感情の吐露 → 共感ポイント → 解決 | いいね・共感 |
| 構造化型 | 問題提起 → 原因（構造） → 実践法 | 保存・フォロー |
| ストーリー型 | 過去→転換点→現在 | いいね・信頼 |
| ランキング型 | 自分なりランキング → 総括 | 自己照合・リプ |

## 感情トリガー一覧

- **自己照合**：「これ自分だ」「あるある」→ いいね・共感
- **損失回避**：「知らないと損」「遅れを取る」→ 保存・クリック
- **逆説驚き**：「思っていたのと違った」→ 拡散・リプ
- **希望・再現性**：「自分でもできるかも」→ フォロー・クリック
- **知的緊張**：「本当に理解しているか？」→ 保存・リプ
- **孤独・共感**：「誰にも言えなかった」→ いいね・感情反応

## 文字数・改行ルール

- 最適文字数：**140〜280字**
- 1行の文字数目安：**15〜25字**
- 改行：**1〜2文ごと**に改行してスキャンしやすくする
- 箇条書きは各項目を1行に収める
- ブロック間に空白行を入れるとリズムが出る

## バズ投稿チェックリスト

投稿前に以下を確認すること。

### 冒頭チェック
- [ ] 1行目が「自己照合」「損失回避」「逆説」のどれかを引き起こしているか
- [ ] 冒頭の型（数字・問題提起・あるある・結論先出し等）を意識して選んでいるか
- [ ] 「続きを読みたい」と思わせる引きがあるか

### 構成チェック
- [ ] 構成パターン（箇条書き・ビフォーアフター・問いかけ等）が明確か
- [ ] 1投稿1目的（いいね・リプ・保存・クリックのどれか）になっているか
- [ ] 感情トリガー（自己照合・損失回避・逆説等）が1つ以上入っているか

### 文体チェック
- [ ] 1行15〜25字・1〜2文ごとに改行されているか
- [ ] 文字数は140〜280字の範囲か
- [ ] ハッシュタグは感情系0個・ノウハウ系1〜2個にとどめているか

### 結びチェック
- [ ] 結び方の型（保存促進・リプライ誘発・共感・行動促進等）を使っているか
- [ ] 「いいことを言っているだけ」で終わっていないか

### アカウント軸チェック
- [ ] 実績がない場合、「観察・気づき・仮説」語りになっているか（成功を語っていないか）
- [ ] 「思考を取り戻す仕事術」の軸から外れていないか
- [ ] 今のフォロワー規模に対して不自然な主張をしていないか
