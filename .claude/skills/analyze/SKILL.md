# Analyze Skill

この skill は WordPress・note・X の運用状況を分析し、優先アクションを提案するためのものです。

## 目的
- 現在の数値を目標と比較してギャップを把握する
- 各媒体のボトルネックを特定する
- 今週やるべきアクションを優先順位付きで提示する
- 導線（X→WordPress→note）の機能状態を確認する

## 事前準備（自動取得）
分析前に以下を実行すると metrics.md が最新の状態になる。

```bash
pip install -r scripts/requirements.txt  # 初回のみ
python scripts/fetch_metrics.py
```

X のデータも取得したい場合は `.env` に `X_BEARER_TOKEN` を設定すること（`.env.example` 参照）。

## 分析フロー

### 1. 現状スナップショット
各媒体の最新数値を整理して表示する。
- WordPress：記事数・累計PV・直近PV推移
- note：有料記事数・各記事のビュー/スキ/売上
- X：フォロワー数・直近の投稿反応

### 2. 目標ギャップ分析
roadmap.md の短期目標（直近の締め切り）と現在数値を比較し、
残り日数に対して「このペースで届くか」を判定する。

判定基準：
- ✅ 順調：現ペース維持で達成可能
- ⚠️ 要加速：現ペースでは未達、対策が必要
- 🔴 危機：目標に対して大幅に遅れており、戦略の見直しが必要

### 3. ボトルネック特定
以下の観点で各媒体のボトルネックを1つずつ特定する。

**WordPress**
- 記事数は十分か（SEO流入の母数）
- PVが伸びていない記事はあるか
- 有料noteへの導線記事は存在するか

**note**
- ビューが来ているか（流入はあるか）
- ビューがあるのに購入されていない場合 → 記事内容・価格・導線の問題
- ビュー自体が少ない場合 → 集客（X・WordPress）の問題

**X**
- フォロワーが増えていない場合 → 投稿の質・頻度・フック
- いいね・RTはあるがフォローされない → 価値提供が一時的で終わっている
- クリックが少ない → 記事への誘導設計の問題

### 4. 導線の状態確認
X → WordPress → note の流れが機能しているかを確認する。
- Xの価値提供投稿でWordPressへの流入が発生しているか
- WordPressの記事内にnoteへの導線が設置されているか
- noteのビュー数に対して購入率はどの程度か

### 5. 今週のアクション提案
以下の形式で優先順位順に出力する。

🔴 P1（今週必ずやる）
- 具体的なアクション（何を・どのくらい）

🟡 P2（できればやる）
- 具体的なアクション

🟢 P3（余裕があれば）
- 具体的なアクション

### 6. コンテンツ制作の提案（必要な場合）
不足しているコンテンツがあれば提案する。
- WordPress：書くべき記事テーマと優先順位
- note：作るべき有料記事と価格帯
- X：今週投稿すべき型（刺し/共感/問いかけ/宣伝）の配分

## 出力形式
以下のセクションを順番に出力する。

1. 現状スナップショット（表形式）
2. 目標ギャップ（表形式＋判定）
3. ボトルネック（媒体別・1〜2行）
4. 導線の状態（○/△/✕で評価）
5. 今週のアクション（優先順位付き）
6. コンテンツ制作提案（必要な場合のみ）

## 注意事項
- 数値がない場合は「未記録」と表示し、更新を促す
- 実績のない内容で「成功する」とは言わない
- 現在のアカウント規模（フォロワー29人）に合った現実的な提案をする
- 「思考を取り戻す仕事術」の軸からズレないアクションを提案する
