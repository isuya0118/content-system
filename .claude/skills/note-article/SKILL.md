# Note Article Skill

この skill は note の有料記事を作成するためのものです。

## 目的
- 有料記事として「売れるか」「満足してもらえるか」を両立する
- 忙しい会社員向けに、再現性のある手順やチェックリストを提供する
- 無料記事やX投稿からの導線を受け取り、収益化につなげる
- 300円記事は入口商品、500円記事は本命商品として設計する

## 参照ファイル
作業前に必ず以下を確認すること。
- README.md
- CLAUDE.md
- workflow.md
- templates/note-article.md
- criteria/note-criteria.md
- templates/review-template.md

必要に応じて以下も参照すること。
- strategy/roadmap.md
- strategy/metrics.md
- strategy/links.md

## 最初に確認すること
ユーザーに以下を確認する。
1. 記事タイトル
2. 想定ペルソナ
3. 価格（300円 / 500円 / その他）
4. 記事の役割（入口商品 / 本命商品 / アップセル商品）
5. 主導線元（WordPress / X / 既存note）
6. 無料公開ラインをどこに置くか
7. 既存の関連記事や接続したい有料記事があるか

## 作業フロー
1. まず記事の役割を確認する
   - 入口商品なら「すぐ使える」「買いやすい」を優先
   - 本命商品なら「体系化」「満足度」「アップセル導線」を優先

2. 構成案を作成する
   - 問題提起
   - 原因の整理
   - 体系化された手順
   - まとめ
   - 次の商品 / 関連記事導線
   の流れを基本とする

3. templates/note-article.md に沿って初稿を作成する
   - 導入で「これは自分に必要だ」と思わせる
   - 無料部分で価値を感じさせる
   - 有料部分で再現性と満足度を出す

4. 無料公開ラインを設計する
   - 問題提起 + 一部の再現性までは無料で見せる
   - 価値を感じた後で有料に入るようにする
   - 入口商品は無料範囲を出しすぎない
   - 本命商品は無料範囲でも十分に価値を感じさせる

5. criteria/note-criteria.md に沿って自己レビューする
   - ペルソナに売れるか
   - ペルソナに満足してもらえるか
   - 95点以上か
   を必ず判定する

6. 95点未満なら改善点を優先順位順に出す
   - P1：売れるかに直結する問題
   - P2：満足度に直結する問題
   - P3：表現や導線の改善

7. 必要なら修正版まで出力する

## 出力ルール
- 抽象論で終わらせず、必ず構造と再現性を示す
- 「知ってよかった」ではなく「使える」と感じる内容にする
- 実体験は補強として使い、主軸は構造化した説明にする
- 実績がない場合は、成功談ではなく観察・仮説・気づきとして提示する
- 「弱い断定」は避け、「強い仮説提示」にする
- タイトル、導入、無料公開ライン、CTAまで一貫して設計する
- 売り込みは強くしすぎず、「続きを読む理由」を作る
- 300円記事から500円記事に送るアップセル導線を意識する

## 価格ごとの基本方針

### 300円記事
- 入口商品
- すぐ使える
- 読む負担が軽い
- チェックリスト、最低限の観点、短い手順が向いている
- 有料ラインは早すぎず、2つ程度価値を見せた後が基本

### 500円記事
- 本命商品
- 体系化された手順
- 「なぜそうするか」まで説明する
- 実務でそのまま使える状態を目指す
- 導線はWordPressの深い無料記事から受ける

## 出力形式
必要に応じて以下を分けて出力する。
- タイトル案
- 構成案
- 初稿
- 無料公開ライン案
- レビュー結果
- 修正版
