# Review Content Skill

この skill は、既存コンテンツをレビューするためのものです。

## 目的
- 既存の draft を媒体ごとの criteria に沿って採点する
- 95点未満の場合は、改善点を優先順位順に明示する
- 必要なら修正版まで提示する
- 「良い」ではなく「公開してよいか」を判定する

## 参照ファイル
作業前に必ず以下を確認すること。
- README.md
- CLAUDE.md
- workflow.md
- templates/review-template.md

媒体に応じて以下を確認すること。
- criteria/wordpress-criteria.md
- criteria/note-criteria.md
- criteria/x-criteria.md

必要に応じて以下も参照すること。
- strategy/roadmap.md
- strategy/metrics.md
- strategy/links.md

## 最初に確認すること
ユーザーに以下を確認する。
1. 対象媒体
   - WordPress
   - note
   - X
2. 対象ファイル名または本文
3. 想定ペルソナ
4. 価格（noteの場合）
5. 誘導先URLや関連する有料 / 無料記事があるか
6. レビューだけ欲しいのか、修正版まで欲しいのか

## 作業フロー
1. 対象媒体を判定する
   - WordPress / note / X のどれかを明確にする

2. 媒体に応じた criteria を選ぶ
   - WordPress → criteria/wordpress-criteria.md
   - note → criteria/note-criteria.md
   - X → criteria/x-criteria.md

3. templates/review-template.md に沿ってレビューする
   必ず以下を含める。
   - 現在点数
   - 良い点
   - 95点未満の理由
   - 修正すべきポイント（優先順位順）
   - 公開可否

4. 95点未満なら改善方針を提示する
   - P1：売れるか / 刺さるか / 伸びるか に直結する問題
   - P2：構成・再現性・満足度に直結する問題
   - P3：文体・表現・導線の改善
   の順に整理する

5. ユーザーが希望した場合は修正版まで出力する
   - 全文修正版
   - 差し替え箇所のみ
   - タイトル修正案
   - 導線修正案
   のいずれかを適切に提示する

## 出力ルール
- 「悪くない」で済ませず、公開基準で判断する
- 95点以上かどうかを必ず明示する
- ペルソナに対して刺さるか / 売れるか / 満足するかを重視する
- WordPress は「無料記事として有料級か」「有料noteへの導線が自然か」を重視する
- note は「売れるか」「購入後に満足してもらえるか」を重視する
- X は「伸びるか」「反応が取れるか」「導線が機能するか」を重視する
- 実績がない場合は、成功者目線ではなく観察・仮説・解像度で評価する
- 抽象的な感想で終わらず、必ず修正可能な指摘にする

## 媒体ごとの重点観点

### WordPress
- 無料記事として単体で価値があるか
- 「思考を取り戻す仕事術」の軸に回収されているか
- 有料noteへの橋渡しが自然で強いか

### note
- 想定ペルソナに対して売れるか
- 価格に見合う具体性と再現性があるか
- 読後に満足し、次の商品にも進みたくなるか

### X
- 1行目で止まるか
- いいね / リプ / クリックのどれを狙うか明確か
- 今のアカウント規模でも成立するか

## 出力形式
必要に応じて以下を分けて出力する。
- レビュー結果
- 採点
- 改善点一覧
- 修正版
- 差し替え文案
