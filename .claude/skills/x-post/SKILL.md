# X Post Skill

この skill は X の投稿を作成するためのものです。

## 目的
- 認知を広げる
- 共感・保存・リプライを獲得する
- WordPress記事またはnote記事への流入を作る
- 「思考を取り戻す仕事術」の軸で、一貫した発信を行う

## 参照ファイル
作業前に必ず以下を確認すること。
- README.md
- CLAUDE.md
- workflow.md
- templates/x-post.md
- criteria/x-criteria.md
- templates/review-template.md

必要に応じて以下も参照すること。
- strategy/roadmap.md
- strategy/metrics.md
- strategy/links.md

## 最初に確認すること
ユーザーに以下を確認する。
1. 投稿の目的
   - 認知拡大
   - 共感獲得
   - リプライ獲得
   - 無料記事誘導
   - 有料記事誘導
2. 想定ペルソナ
3. テーマ
4. 誘導先URL（ある場合）
5. 強めに刺すか、ややマイルドにするか
6. 投稿本数（通常は3〜10案）

## 作業フロー
1. 投稿の役割を明確にする
   - この投稿で狙うのは「いいね / リプ / クリック」のどれかを明示する
   - 1投稿1目的で設計する

2. templates/x-post.md に沿って複数案を作成する
   - 刺し投稿
   - 共感投稿
   - 問いかけ投稿
   - 宣伝投稿
   - リプライ用追撃投稿
   の中から、目的に合う型を選ぶ

3. 文字数を調整する
   - 日本語のX投稿として自然な長さにする
   - 原則、冗長な説明を避ける
   - 1行目で止まる設計にする

4. criteria/x-criteria.md に沿って自己レビューする
   - 伸びる可能性があるか
   - 反応が取れるか
   - クリック導線として機能するか
   を必ず判定する

5. 95点未満なら改善案を優先順位順に出す
   - P1：1行目の弱さ
   - P2：共感 / 問い不足
   - P3：クリック理由不足
   - P4：売り込み感の強さ
   を優先して修正する

6. 必要なら修正版まで出力する

## 出力ルール
- 最初の1行で必ず止める
- 「いいことを言う」のではなく「反応を起こす」投稿にする
- 抽象論で終わらせず、違和感・構造・具体性を入れる
- 実績がない場合は、成功者目線ではなく観察・仮説・気づきで語る
- 想定ペルソナが「自分のことだ」と思える言葉を入れる
- 誘導投稿では「読まないと損する理由」を作る
- リプライがつきやすいよう、必要に応じて問いで閉じる
- 宣伝比率は高くしすぎず、価値提供3：宣伝1を前提にする
- 「思考を取り戻す仕事術」の軸から外れない

## 投稿タイプごとの基本方針

### 刺し投稿
- 認知拡大向け
- 少し強めでよい
- 読者の痛みや遠回りを言語化する
- へえで終わらせず、少し引っかかる言い回しにする

### 共感投稿
- いいね・保存向け
- 自分の観察や気づきを使う
- ただの日記にしない
- 共感できるズレやあるあるを入れる

### 問いかけ投稿
- リプ獲得向け
- 二択・経験共有・軽い議論が有効
- 正解が1つに決まらない問いにする
- コメントしやすい温度感にする

### 宣伝投稿（無料記事）
- WordPressへの誘導向け
- 読者の課題に刺さるフックを作る
- 記事で分かることを3点以内で示す
- 「まず無料で読む理由」を明確にする

### 宣伝投稿（有料記事）
- noteへの誘導向け
- 無料では解決しきれない課題設定にする
- 「買う理由」を短く明確にする
- 今のアカウント規模で不自然に売り込みすぎない

### リプライ用追撃投稿
- 元ポストのクリック補強向け
- 単体で見られても意味が通るようにする
- 元ポストより少し強めにする
- URLは原則1つのリプにだけ付ける

## 出力形式
必要に応じて以下を分けて出力する。
- 投稿案（3〜10本）
- 最有力案
- 自己レビュー結果
- 修正版
- リプライ案（必要な場合）
