# X Post Skill

この skill は X の投稿を作成するためのものです。

## 目的
- 認知を広げる
- 共感・保存・リプライを獲得する
- WordPress記事またはnote記事への流入を作る
- 「思考を取り戻す仕事術」の軸で、一貫した発信を行う

## 参照ファイル
作業前に必ず以下を確認すること。
- README.md
- CLAUDE.md
- workflow.md
- templates/x-post.md
- criteria/x-criteria.md
- templates/review-template.md
- strategy/x-buzz-patterns.md（バズ投稿の構造・チェックリスト）

必要に応じて以下も参照すること。
- strategy/roadmap.md
- strategy/metrics.md
- strategy/links.md

## 最初に確認すること
ユーザーに以下を確認する。
1. 投稿の目的
   - 認知拡大
   - 共感獲得
   - リプライ獲得
   - 無料記事誘導
   - 有料記事誘導
2. 想定ペルソナ
3. テーマ
4. 誘導先URL（ある場合）
5. 強めに刺すか、ややマイルドにするか
6. 投稿本数（通常は3〜10案）

## 作業モード

### 通常モード（記事誘導・価値提供投稿）
1. 投稿の役割を明確にする
   - この投稿で狙うのは「いいね / リプ / クリック」のどれかを明示する
   - 1投稿1目的で設計する

2. templates/x-post.md に沿って複数案を作成する
   - 刺し投稿
   - 共感投稿
   - 問いかけ投稿
   - 宣伝投稿
   - リプライ用追撃投稿
   の中から、目的に合う型を選ぶ

3. 文字数を調整する
   - 日本語のX投稿として自然な長さにする
   - 原則、冗長な説明を避ける
   - 1行目で止まる設計にする

4. criteria/x-criteria.md に沿って自己レビューする
   - 伸びる可能性があるか
   - 反応が取れるか
   - クリック導線として機能するか
   を必ず判定する

5. 95点未満なら改善案を優先順位順に出す
   - P1：1行目の弱さ
   - P2：共感 / 問い不足
   - P3：クリック理由不足
   - P4：売り込み感の強さ
   を優先して修正する

6. 必要なら修正版まで出力する

### バズ狙いモード（1000いいね以上を目指す投稿）

ユーザーから「バズらせたい」「伸ばしたい」という指示があった場合はこのモードで作業する。

#### ステップ1：バズ投稿リサーチ（30例）
WebSearchを使い、以下テーマで1000いいね以上のバズ投稿パターンを30例収集する。

調査対象テーマ（ユーザーの軸に合わせて選ぶ）：
- 副業・会社員副業
- AI活用・仕事術
- Claude Code

検索クエリ例：
- "副業" バズ 投稿 1000いいね 構造
- "AI 仕事術" バズ ツイート 日本語
- "Claude Code" バズ 投稿例
- バズる投稿 型 テンプレート X 2025

収集する情報：
- 冒頭1〜2行のパターン
- 構成の型（箇条書き・対比・ストーリー等）
- 感情トリガー（自己照合・損失回避・逆説・共感など）
- 文字数・改行スタイル
- ハッシュタグの有無と数
- 結び方のパターン

#### ステップ2：なぜなぜ分析
収集した30例を横断分析し、なぜバズるのかを5段階で掘り下げる。

分析軸：
- なぜ止まって読むのか（L1）
- なぜそれが機能するのか（L2：心理メカニズム）
- なぜこのテーマで特に機能するのか（L3：ペルソナの状態）
- なぜ保存・共有されるのか（L4：行動動機）
- なぜ1000いいねになるのか（L5：拡散の構造）

#### ステップ3：バズ投稿5案の作成
なぜなぜ分析の結果を元に、ユーザーのアカウント（思考を取り戻す仕事術・忙しいSE向け）に合った投稿を5案作成する。

各案に以下を明記する：
- 型の名前（SEあるある型・逆説フック型・ビフォーアフター型 等）
- 狙うエンゲージメント（いいね / リプ / 保存 / 拡散）
- 投稿本文

制約：
- 実績がない場合は「観察・気づき・仮説」語りにする（成功を語らない）
- 1行15〜25字・1〜2文ごとに改行する
- ハッシュタグは感情系0個・ノウハウ系1〜2個にとどめる
- 「思考を取り戻す仕事術」の軸から外れない

#### ステップ4：criteria/x-criteria.md でレビュー
各案を採点し、95点未満の場合は修正版を出す。

## 出力ルール
- 最初の1行で必ず止める
- 「いいことを言う」のではなく「反応を起こす」投稿にする
- 抽象論で終わらせず、違和感・構造・具体性を入れる
- 実績がない場合は、成功者目線ではなく観察・仮説・気づきで語る
- 想定ペルソナが「自分のことだ」と思える言葉を入れる
- 誘導投稿では「読まないと損する理由」を作る
- リプライがつきやすいよう、必要に応じて問いで閉じる
- 宣伝比率は高くしすぎず、価値提供3：宣伝1を前提にする
- 「思考を取り戻す仕事術」の軸から外れない

## 投稿タイプごとの基本方針

### 刺し投稿
- 認知拡大向け
- 少し強めでよい
- 読者の痛みや遠回りを言語化する
- へえで終わらせず、少し引っかかる言い回しにする

### 共感投稿
- いいね・保存向け
- 自分の観察や気づきを使う
- ただの日記にしない
- 共感できるズレやあるあるを入れる

### 問いかけ投稿
- リプ獲得向け
- 二択・経験共有・軽い議論が有効
- 正解が1つに決まらない問いにする
- コメントしやすい温度感にする

### 宣伝投稿（無料記事）
- WordPressへの誘導向け
- 読者の課題に刺さるフックを作る
- 記事で分かることを3点以内で示す
- 「まず無料で読む理由」を明確にする

### 宣伝投稿（有料記事）
- noteへの誘導向け
- 無料では解決しきれない課題設定にする
- 「買う理由」を短く明確にする
- 今のアカウント規模で不自然に売り込みすぎない

### リプライ用追撃投稿
- 元ポストのクリック補強向け
- 単体で見られても意味が通るようにする
- 元ポストより少し強めにする
- URLは原則1つのリプにだけ付ける

## 投稿作成フロー（ideas → drafts → reviews）

### 1. ideas（アイデア出し）
`contents/03_X/ideas/YYYY-MM-DD_slug.md` にアイデアを保存する。

記載内容：
- 投稿の目的（いいね / リプ / 保存 / クリック）
- 狙うテーマ・切り口
- バズパターンの型（strategy/x-buzz-patterns.md を参照）
- 誘導先URL（ある場合）
- 投稿本数の目安

### 2. drafts（初稿作成）
`contents/03_X/drafts/YYYY-MM-DD_slug.md` に投稿本文を作成する。

- strategy/x-buzz-patterns.md のチェックリストを参照しながら書く
- 複数案（3〜10本）を出す
- 1行目で止まる設計を最優先にする

### 3. reviews（レビュー）
`contents/03_X/reviews/YYYY-MM-DD_slug.md` にレビュー結果を保存する。

以下の順でレビューする：
1. strategy/x-buzz-patterns.md のバズ投稿チェックリスト
2. criteria/x-criteria.md の全チェック項目

レビュー結果には以下を明記する：
- 現在点数
- 95点未満の理由
- 修正すべきポイント（P1〜P3）
- 最有力案の選定理由

### 4. revise → 公開
95点以上になったら公開。投稿済みのものは `contents/03_X/published/` に記録する。
